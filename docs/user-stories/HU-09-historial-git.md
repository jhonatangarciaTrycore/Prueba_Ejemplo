# HU-09 — Historial Git correcto

**Como** revisor
**Quiero** que el repositorio tenga ≥3 commits con mensajes descriptivos
**Para** evitar entregas con commits masivos o poco informativos.

## Requisitos
- Mínimo 3 commits durante la implementación.
- Mensajes en imperativo y descriptivos (ej.: "Add match endpoint", "Add unit tests for EXCELLENT_FIT").
- No usar mensajes genéricos: `wip`, `fix`, `cambios`.

## Criterios de aceptación
- `git log` muestra ≥3 commits con mensajes adecuados.
