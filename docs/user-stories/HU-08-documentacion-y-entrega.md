# HU-08 — Documentación y entrega

**Como** evaluador
**Quiero** un `README.md` en la raíz con instrucciones claras
**Para** poder instalar, ejecutar y testear sin ambigüedades.

## Contenido mínimo del README
- Comando exacto para instalar dependencias y correr el servicio.
- Comando exacto para ejecutar los tests.
- Sección "Decisiones" donde se documenten ambigüedades resueltas (p. ej. `skills_requeridas` vacío).
- Nota sobre el stack elegido (si no es Java/Python/Node, una línea con la razón).

## Criterios de aceptación
- README presente en la raíz con las secciones mencionadas.
