# HU-07 — Pruebas unitarias obligatorias

**Como** QA
**Quiero** una suite de tests que cubra los casos solicitados
**Para** asegurar que la lógica cumple los criterios de aceptación.

## Tests mínimos requeridos
- EXCELLENT_FIT (todas las skills, experiencia ≥ mínima, contrato `indefinido`).
- GOOD_FIT.
- MAYBE.
- NO_FIT (cobertura baja o experiencia muy lejana).
- Caso borde: `skills_requeridas` vacío (decisión documentada en README).
- Experiencia mayor que `experiencia_min` no debe penalizar.
- Validaciones de input (campos faltantes y tipos incorrectos).

## Criterios de aceptación
- ≥5 tests automáticos (se recomienda usar el runner idiomático del stack elegido).
- Todos los tests pasan localmente con el comando documentado en README.
